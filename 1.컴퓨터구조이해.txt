1. 컴퓨터구조 이해
    1) java, C, C++, python 언어가 프로그램을 실행하는 기반을 제공함.
    2) 소프트웨어가 실행되는 영역은 하드웨어(>CPU)임.
    3) 소프트웨어를 실행한다는 의미는 
       프로그래머가 미리 만들어 둔 명령어를 하드웨어의 메인인 
       중앙처리장치(CPU, Central Processint Unit)가 
       그 명령어대로 컴퓨터의 하드웨어를 작동시키는 과정이다.

2. 중앙처리장치 
    1) 컴퓨터에서 프로그램의 명령어를 수행하는 컴퓨터의 두뇌와 같은 역할을 함.
    2) 구성요소
        -산술(사칙연산)/논리(참 or 거짓)부
        -제어부(true or false)
        -레지스터(메모리)

3. 주메모리(Main Memory=RAM)
    1) RAM
    2) 컴퓨터가 작업을 수행하는 동안 작업 수행에 필요한 명령어와 자료(Data)를 저장하는 공간
    3) 주메모리는 내부적으로 자료를 저장할 수 있는 셀(cell)로 이루어져 있음.
    4) 각각의 셀에는 고유번호(=주소, address)가 할당되어 있음

4. 소프트웨어
    1) 컴퓨터 하드웨어의 기능을 통제하여 원하는 결과나 기능을 얻도록 하는 명령어의 집합을 의미함.
    2) 종류
        -시스템 소프트웨어(System software)
        -응용 소프트웨어(Application software)
         등등
    3) 시스템 소프트웨어(System software)
        -운영체제(OS): 시스템의 효율적인 운영과 관리를 목적으로 하는 소프트웨어
        -컴파일러(0과 1로 변환시켜주는 소프트웨어), 디버거(버그제거): 다른 소프트웨어를 개발하는데 사용되는 소프트웨어(로 프로그램에 내장되어 있음)
    4) 응용 소프트웨어(Application software)
        -프로그램: 오피스 프로그램, 게임, 브라우저...

5. 프로그램 개발 과정
    1) 문법에 맞게 소스 코드를 작성, 파일로 저장
    2) 2진 명령어 코드로 변환 작업인 컴파일을 함
    3) 실행코드(실행 파일)를 실행 => CPU는 2진 코드를 읽어서 실행함
    4) 통합 개발 환경(IDE, Integrated Development Environment) 제공
        -이클립스, 인텔리J: 이 tool안에는 편집기, 컴파일러, 디버거, 인터프리터 등이 포함되어 있음.

6. 프로그래밍 언어
    1) 초기 프로그래밍
        -프로그램은 미리 정해진 일을 순서대로 컴퓨터 하드웨어에 전기 신호를 보내서 동작시킴.
        -전류를 약하게 보냈다(0)가 강하게 보냈다(1)가 하는 방법
        -이런 처리 절차를 하나하나 수동으로 적었음
        -0과 1로 표현하는 프로그래밍 언어를 기계어라고 함
        -2의 8승(두 종류의 비트(0과 1)를 8개의 단위(바이트)로 묶음), 즉 256개 동작을 구분할 수 있음
        -경우의 수가 너무 많아져 기계어에 일대일 대응하는 단어를 이용한 프로그래밍 방식이 나옴=>어셈블리어
        -상대적으로 저수준 언어(=기계에 더 친화적인 언어)
    2) 현재 프로그래밍
        -고수준 언어(=사람에 더 친화적인 언어) 출현
